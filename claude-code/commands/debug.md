# Debug Issue

버그나 이슈를 체계적으로 디버깅하고 수정해주세요.

**디버깅 프로세스:**

1. **문제 재현**
   - 증상 확인
   - 재현 단계 파악
   - 에러 메시지/로그 분석

2. **원인 분석**
   - 관련 코드 탐색
   - 실행 흐름 추적
   - 상태 및 데이터 확인
   - 가설 수립

3. **문제 격리**
   - 최소 재현 케이스 생성
   - 관련 컴포넌트/함수 식별
   - 의존성 확인

4. **해결 방안**
   - Root cause 수정
   - Edge case 처리
   - 방어적 코딩 추가

5. **검증**
   - 수정 사항 테스트
   - 회귀 테스트
   - 다른 영향 확인

6. **예방**
   - 테스트 케이스 추가
   - 유사 패턴 검토
   - 문서화

**도구 활용:**
- Console.log (전략적 배치)
- Debugger
- Network 탭 (API 호출)
- React DevTools
- Prisma Studio (DB 확인)
